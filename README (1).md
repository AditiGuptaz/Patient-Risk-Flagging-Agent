# Patient Risk Flagging Agent

A single-agent system built with LangGraph that reads a raw clinical note, checks medication safety against kidney function, scores patient risk, and escalates dangerous cases to a human reviewer before producing a final report.

Built as a first hands-on agentic AI project, applying a healthcare analytics background to the design of the safety logic.

## What This Project Does

Given one patient's clinical note as plain text, the agent:

1. Extracts structured facts from the note using an LLM (age, diagnosis, medications, labs, visit history)
2. Retrieves relevant clinical guideline context for that diagnosis
3. Checks whether any current medication is unsafe given the patient's kidney function (eGFR)
4. Scores the patient's overall risk as Low, Medium, or High
5. If the case is High risk or a safety issue was found, pauses for human review before continuing
6. Produces a structured summary with risk level, reasoning, and recommended action
7. Logs the outcome to an audit trail

It is a single agent — one continuous decision-making process — rather than a team of agents handing work to each other.

## Architecture

```
START
  │
  ▼
Extract Patient Data  (LLM call — parses raw note into structured fields)
  │
  ▼
Retrieve Guidelines  (ChromaDB vector search over a clinical guideline PDF)
  │
  ▼
Medication Safety Check  (flags renally-cleared drugs vs. eGFR)
  │
  ▼
Risk Scoring  (combines safety flags, HbA1c, care gap → Low / Medium / High)
  │
  ▼
Safety flag present, or risk = High?
  │
  ├── YES ──▶ Human Review Checkpoint ──┐
  │                                      │
  └── NO ───────────────────────────────▶│
                                          ▼
                              Generate Structured Output
                                          │
                                          ▼
                                  Log to Audit Trail
                                          │
                                          ▼
                                         END
```

## Tech Stack

- **LangGraph** — stateful graph orchestration, conditional routing, human-in-the-loop checkpoint
- **LangChain + OpenAI (gpt-4o-mini)** — clinical note extraction with structured JSON output
- **ChromaDB + OpenAI embeddings (text-embedding-3-small)** — semantic search over a clinical guideline PDF, retrieving relevant guidance by meaning rather than exact keyword matches
- **Python** — rule-based medication safety and risk-scoring logic


Then run the agent:

```python
from graph import app
import json

with open("mock_patients.json") as f:
    data = json.load(f)

result = app.invoke({
    "patient_id": data["patients"][2]["patient_id"],
    "raw_clinical_note": data["patients"][2]["raw_clinical_note"]
})
print(result["final_output"])
```

`guideline_context` is retrieved through real semantic search against `clinical_guidelines.pdf` — the agent isn't told which guideline applies, it finds the relevant one(s) by meaning. For a CHF + diabetes patient, it correctly surfaces both relevant sections, not just one.

The two high-risk patients triggered the human checkpoint. The four low/medium-risk patients passed straight through. The agent extracted every value itself from unstructured text — none of this was hand-fed.

## Current Limitations

This is a first build, and being upfront about what's simplified matters as much as what works:

- **Guideline chunking relies on detecting known heading text** (`KNOWN_HEADINGS` in `build_vectorstore.py`), rather than a general-purpose document parser. This guarantees clean, complete chunks for *this* PDF, but adding a new guideline document with different heading conventions would need that list updated — it isn't fully automatic yet.
- **The human review checkpoint** logs the case and auto-approves rather than genuinely pausing execution. A production version would use LangGraph's `interrupt()` with a persistent checkpointer to wait for a real person's decision.
- **Tested on 6 synthetic patients only.** This demonstrates the architecture and reasoning pattern, not clinical validation against real-world data.

## Why I Built This

I work as a healthcare data analyst and am building toward AI/ML roles at the intersection of healthcare and agentic AI. This project was a deliberate choice to combine both: the technical pattern (LangGraph, structured extraction, conditional routing) and the domain judgment (knowing that Metformin and eGFR below 30 is a real, specific danger — not a detail a general-purpose AI engineer would think to check for).

The most important design decision in the whole project isn't a model choice — it's the line that decides when a human has to step in before the agent goes any further.
